- **D-30 · RPC `fuetterung_erfassen` als einziger Schreibpfad** (wie 4.5): `distinct` Völker → je 1 Zeile, Lager-Abbuchung `menge_pro_volk_kg × ROW_COUNT` (nie `array_length`), `betrieb_id` explizit. **Enums (`zweck`/`futterart`) IN der RPC geprüft** → BA040 statt rohem 23514 (Review M9). Keine INSERT-Policy (Insert RPC-only), keine DELETE-Policy (Soft-Delete). BA040 Pflichtfeld/Enum, BA041 Völker, BA042 Material-Tenancy.
- **D-31 · Winterfutter-Ziel + Saison-Σ:** `winterfutter_ziel_kg` in `betriebs_einstellungen` (F4-Parameter, Default 22, CHECK > 0), mandantenfähig. **Nur `auffuetterung` zählt** fürs Ziel (Reiz/Not nicht). **Saison-Anker in `winterfutterKg` gekapselt (Review M2):** `Monat < 7 → Vorjahr` — sonst hätte `now().year` den Balken Jan–Juni auf 0 genullt (mitten in der Überwinterung), weil die Auffütter-Saison über den Jahreswechsel läuft. Fenster-Start `kAuffuetterSaisonStartMonat = 7` (Nordhalbkugel-Fachdefault, F4 später).
- **Gotcha (App):** (1) `menge_pro_volk_kg` = **Produktmasse je Volk** (Review M3 — Spalte/Label mit „pro Volk", sonst ab dem 2. Volk vervielfacht die RPC-`× v_n`-Abbuchung; M5 — Balken labelt „Produktgewicht ≠ Vorrat"). (2) Finaler Review: Balken-`(prozent*100).floor()` statt `round()` (sonst „100 %" neben amber/„nicht erreicht" bei 21.9/22); **Bio-Schalter Default `false` (fail-safe)** — `true` hätte bei vergessenem Umlegen einen stillen `bio_zertifiziert=true`-Falsch-Positiv im Bio-Nachweis erzeugt. (3) `betriebsEinstellungenProvider` steht schon seit 4.2 in `_datenNeuLaden` (der Balken liest daraus das Ziel); nur `fuetterungenFuerVolkProvider` neu ergänzt.
